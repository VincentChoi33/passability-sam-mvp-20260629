시작 중

이동조건

입력값은 기준속도이고, 실제 ETA는 경사/노면/진흙/신뢰도 난이도 보정을 적용합니다.

배경설정

관심영역은 S/D 중앙점 기준 5km x 5km 계산 범위이고, 표시 강도만 낮춰 배경지도를 더 잘 보이게 할 수 있습니다.

먼저 도로 경로를 구한 뒤, S/D 중앙점 기준 5km x 5km를 SAM3.1 판독 범위로 잡고 그 안에서 경로 wrapper를 평가합니다.

SAM3.1 경로 판독 서버 대기

SAM3.1 영상분할은 S/D 중앙점 기준 5km 영역별로 새로 계산되어야 합니다. 현재 5km bbox와 10m grid 조건을 develop 서버에 보내고, 돌아온 segmentation grid를 네이버 경로 위에 덮어 위험도를 평가합니다.

차단

차단구역 지정 모드에서 지도를 드래그하면 지정한 사각 범위대로 검정 차단구역이 생성됩니다. 그냥 클릭하면 기본 지정 크기로 생성됩니다. 차단구역 해제는 클릭한 지정구역 전체를 삭제합니다.

격자를 클릭하면 판단 이유를 볼 수 있습니다. 출발지/도착지는 가장 가까운 10m 격자에 맞춰집니다.

계산 논리

지도가 깔리는 방식

화면의 배경은 일반 지도 또는 위성사진 타일입니다. 지도 전체를 한 장으로 받는 것이 아니라, 현재 화면 위치와 줌 레벨에 맞는 작은 정사각형 이미지 조각들을 여러 장 받아 이어 붙입니다. 이 조각을 지도 타일이라고 부릅니다.

OpenStreetMap 타일

OpenStreetMap은 도로, 지명, 하천, 건물 같은 공개 지도 데이터를 기반으로 만든 지도입니다. 타일은 줌 레벨, 가로 번호, 세로 번호로 구분됩니다. 사용자가 지도를 움직이거나 확대하면 브라우저가 필요한 번호의 타일만 새로 받아옵니다. 그래서 전국 지도를 한 번에 내려받지 않고도 필요한 화면만 빠르게 보여줄 수 있습니다.

위성사진 타일

기본 배경은 외부 위성사진 백업 타일입니다. 브라우저는 현재 화면 위치와 줌 레벨에 맞는 위성사진 타일을 받아 지도처럼 이어 붙입니다. 네이버 위성지도도 배경 선택지로 남겨두지만, 길찾기는 배경지도와 별개로 Naver Directions 도로 후보를 서버에서 받아와 우리 10m 격자로 재평가합니다.

Leaflet의 역할

Leaflet은 웹브라우저에서 지도를 다루기 위한 JavaScript 지도 라이브러리입니다. 이 MVP에서는 OpenStreetMap 타일을 화면에 깔고, 마우스 드래그로 지도를 이동시키고, 줌 버튼으로 확대/축소하고, 위도/경도 좌표를 화면 좌표로 바꾸고, 출발지/도착지 마커와 경로선을 지도 위에 올리는 일을 맡습니다.

지도와 격자의 관계

Leaflet은 배경지도와 화면 조작을 담당하고, 통행성 계산은 별도의 10m 격자에서 합니다. 출발지 S와 도착지 D를 잡고 경로 계산을 누르면 먼저 도로 경로를 받아온 뒤, 그 경로가 지나가는 영역에 여유 폭을 붙여 관심영역을 만듭니다. 이후 그 영역을 10m 단위 칸으로 나누고, 각 격자 칸을 Leaflet 지도 좌표에 맞춰 반투명 색상으로 덮어 보여줍니다.

영상/DEM 판독 MVP

이번 버전은 외부 위성사진 백업 배경 위에 DEM 경사와 영상분할 결과를 같은 10m 격자에 맞춰 올리는 구조입니다. 영상분할 레이어는 실제 모델 출력처럼 각 칸에 도로, 물, 건물, 숲, 초지, 농경지, 나지, 그림자, 불확실 비율과 신뢰도를 붙입니다. 현재 브라우저는 판독 결과를 표시하고, 실제 판독값은 develop 서버의 위성영상 export와 SAM3.1 segmentation 결과로 교체됩니다.

수계와 교차점

수계와 소하천은 원칙적으로 차단으로 보고, 도로/산지도로/등산로가 수계를 가로지르는 격자만 crossing 후보로 따로 표시합니다. OSM bridge 태그처럼 교량이 확인되면 초록 점, 암거/수계 위 도로/SAM3.1 후보처럼 제원 확인이 필요한 경우는 주황 점, ford 성격이면 보라 점으로 표시합니다. 교량/암거 후보 표시는 배경설정에서 켜고 끌 수 있습니다.

SAM3.1 판독 결과

현재 샘플 관심영역에는 develop 서버의 SAM3.1 weight로 만든 segmentation JSON을 추가로 읽습니다. 네이버 도로 경로가 먼저 정해지면 S/D 중앙점 기준 5km 위성영상을 export하고, 전체 이미지를 한 번 판독한 뒤 768px 타일을 192px씩 겹쳐 다시 판독해서 작은 물줄기와 좁은 선형로를 보강합니다. 이번 판독은 물, 소하천/계류, 일반 도로, 산지도로/임도, 등산로/보행로, 교량/암거 후보, 수목을 따로 저장합니다. 출발지/도착지가 바뀌면 기존 JSON을 잘라 재사용하는 것이 아니라, 새 bbox와 grid 조건으로 위성영상을 다시 export하고 SAM3.1을 다시 실행해야 합니다.

DEM 수문분석 결합

작은 하천은 나무와 그림자 때문에 영상에서 안 보일 수 있어서 DEM도 같이 봅니다. 각 격자는 주변 8방향 중 가장 낮은 칸으로 물이 흐른다고 가정하고, 위쪽에서 모이는 칸 수를 누적해 flow accumulation을 계산합니다. 누적량이 크고 계곡 저지대 형태가 있으면 DEM 추정 소하천/배수로 후보로 표시합니다. SAM이 실제 물을 잡은 곳과 DEM 흐름선이 겹치면 확정 수계 신뢰도를 올리고, DEM만 잡은 곳은 바로 차단하지 않고 고비용 위험 구간으로 둡니다.

격자가 깔리는 이유

S/D 중앙점 기준 5km x 5km 관심영역을 10m 격자로 쪼개 각 칸마다 경사, 도로 여부, 수계, 건물, 지표분류, 강우 영향을 계산합니다. 현재 MVP는 항상 500 x 500, 총 25만 칸을 계산합니다. 네이버 경로가 이 5km 박스 밖으로 나가면 경로 일부가 판독영역 밖이라고 표시하고, 긴 경로는 서버 타일링 방식으로 나누는 구조가 다음 단계입니다. 이후 이동 주체별로 같은 칸도 다르게 평가합니다. 전국 단위는 10m 격자가 약 10억 칸 규모라 브라우저 한 화면 계산이 아니라 타일/캐시 서버 방식으로 나눠 처리해야 합니다.

색상 판정

  • 초록: 해당 이동 주체가 비교적 무리 없이 통과 가능한 칸
  • 노랑: 통과는 가능하지만 경사, 비도로, 지표 미분류, 낮은 신뢰도 같은 주의 조건이 있는 칸
  • 주황: 수계/건물 같은 확정 차단은 아니지만 비용 점수가 높아 경로가 강하게 회피하는 칸
  • 빨강: 건물, 확인되지 않은 수계 crossing, 허용 경사/폭 초과, 도로폭 부족처럼 통과 불가 규칙에 걸린 칸
  • 검정: 사용자가 직접 지정한 현장 차단구역

색상판정 근거

각 10m 격자는 먼저 이동 주체별 통과 비용을 계산합니다. 도로 칸은 도로 등급과 경사를 보고 비용을 낮게 주고, 비도로 칸은 경사, 최근 강우량, 진흙 위험, 수목, 농경지, 습지, 나지, 시가지, 지표 미분류, 출처 신뢰도를 반영해 비용을 올립니다. 격자를 클릭하면 비용 점수와 주요 가산 근거가 함께 표시됩니다.

  • 초록: 비용 점수 4.4 이하. 경로 탐색에서 우선 통과 후보로 봅니다.
  • 노랑: 비용 점수 4.4 초과, 13.5 이하. 통과는 가능하지만 속도 저하와 주의 조건을 반영합니다.
  • 주황: 비용 점수 13.5 초과. 원칙적으로 통과 가능하지만 경로 탐색에서는 매우 불리하게 계산합니다.
  • 빨강: 건물/수계/경사초과/도로폭 부족처럼 통과 불가 규칙에 걸린 칸입니다.
  • 검정: 사용자가 현장 상황으로 지정한 차단구역입니다. 자동 계산보다 우선해서 경로 탐색에서 제외합니다.

이동 주체별 차이

같은 격자라도 도보 인원, ATV, 소형 전술차량, 대형 트럭, 궤도형 차량마다 기준이 다릅니다. 예를 들어 도보 인원은 등산로와 급경사를 어느 정도 허용하지만, 차량은 최소 차폭보다 좁은 길을 차단하고 대형 트럭은 산지도로/임도도 폭 기준에 걸릴 수 있습니다. 그래서 이동 주체를 바꾸면 같은 지점의 색상과 경로가 달라질 수 있습니다.

현재 기준의 성격

현재 색상판정은 실제 군 기준표가 아니라 MVP용 설명 가능한 휴리스틱입니다. 실제 적용 단계에서는 차량 제원표, 부대 운용 기준, 최신 DEM, 위성/항공 영상 판독 결과, 교량/암거 제원, 현장 정찰 결과를 넣어 임계값을 교정해야 합니다.

경로가 나오는 방식

경로 계산은 develop 서버의 도로 후보를 먼저 받아오고, 그 후보가 지나는 10m 격자는 도로 네트워크 확인 근거로 보존합니다. 이후 선택한 이동 주체별로 SAM3.1 수계/건물/도로/산지도로 판독, DEM 경사, 사용자 차단구역을 덮어 Naver 후보와 A* 우회 후보를 함께 점수화합니다. 그래서 도보, ATV, 소형 전술차량, 트럭, 궤도형 차량을 바꾸면 같은 S/D라도 1안 경로가 달라질 수 있습니다.

실시간 차단 반영

차단구역 지정 모드에서 차단 종류를 고른 뒤 지도 위에 사각 window를 드래그하면 그 영역이 하나의 지정구역 객체로 저장됩니다. 그냥 클릭하면 기본 지정 크기 window가 생성됩니다. 차단구역 해제 모드에서는 클릭한 칸이 포함된 지정구역 전체를 삭제합니다.

예상시간 계산

사용자가 입력한 도로/비도로 기준속도에 도로 등급, 경사, 노면, 진흙위험, 신뢰도, 난이도 점수를 반영해 칸별 속도를 낮춥니다. 그 칸별 이동시간을 합산한 값이 예상 이동시간입니다.

현재 데이터는 OSM, 위성사진 타일, Terrain-RGB DEM, 샘플 영상분할을 조합한 데모 계산입니다. 실제 운용에는 고해상도 DEM, 최신 정사영상, 실제 segmentation 모델, 교량/도로 제원, 현장 검증 데이터가 추가로 필요합니다.
